api-res-kit

Standardize your Node.js API responses with a consistent, type-safe format. Works with Express, Fastify, and NestJS.

The Problem

Your API responses are a mess

Every endpoint returns a different shape. No consistency, no standard, no predictability.

your-api.ts — spot the pattern (there isn't one) 
// Endpoint A — returns "status" + "result"
res.json({ status: true, result: { id: 1 } })

// Endpoint B — returns "success" + "data"
res.json({ success: true, data: { id: 1 } })

// Endpoint C — returns "ok" + "payload"
res.json({ ok: 1, payload: { id: 1 } })

// Endpoint D — completely different error shape
res.status(404).json({ error: "Not found" })

// Endpoint E — yet another format
res.json({ code: 200, msg: "ok", body: {} })

Frontend Can't Predict the Shape

Is the data in result, data, payload, or body? Frontend devs write different parsing logic for every single endpoint.

Mobile Apps Break Silently

One backend dev changes the response format, and the iOS and Android apps crash in production with no warning.

New Devs Waste Hours

No documentation, no convention. Every new team member has to reverse-engineer each endpoint to understand the response format.

API Integrations Fail

Third-party consumers can't build reliable integrations when your error format changes between endpoints.

The Solution

One format. Every response.

Drop in api-res-kit and every response — success, error, validation, paginated — follows the same predictable structure.

success

Boolean flag — instantly know if the request succeeded or failed. No guessing.

message

Human-readable status message. Show it directly in your UI toast or alert.

data

The actual payload — always in the same key. Users, posts, tokens — it's always data.

errors

Field-level error array with codes. Wire directly to form validation on the frontend.

meta

Extensible metadata — pagination info, request IDs, timestamps, API version.

response.json — every response, same shape 
{
  "success": true,
  "message": "User created successfully",
  "data": {
    "id": 1,
    "name": "Alice",
    "email": "alice@example.com"
  },
  "errors": [],
  "meta": {}
}
Features

Everything you need

Built for real-world backend development. Type-safe, framework-agnostic, zero dependencies.

Success Responses

success(), created(), noContent() — standard 2xx responses with consistent format.

Error Responses

error(), notFound(), unauthorized(), forbidden() — every error follows the same shape.

Validation Errors

Field-level validation errors with custom codes. Perfect for form feedback.

Pagination

Auto-calculated totalPages, hasNextPage, hasPrevPage in the meta object.

TypeScript First

Full generic support. Your data types flow through the response builders.

Zero Dependencies

Lightweight and fast. No bloat, no supply chain risk. Just pure utility.

API Reference

Simple, intuitive functions

Clean, type-safe functions for every response scenario.

success(res, options?)

200 
import { success } from "api-res-kit"

app.get("/users", async (req, res) => {
  const users = await User.find()

  success(res, {
    message: "Users fetched",
    data: users
  })
})

created(res, options?)

201 
import { created } from "api-res-kit"

app.post("/users", async (req, res) => {
  const user = await User.create(req.body)

  created(res, {
    message: "User created",
    data: user
  })
})

error(res, options?)

500 
import { error } from "api-res-kit"

app.get("/crash", (req, res) => {
  error(res, {
    message: "Database failed",
    code: 500
  })
})

notFound(res, message?)

404 
import { notFound } from "api-res-kit"

app.get("/users/:id", async (req, res) => {
  const user = await User.findById(req.params.id)
  if (!user)
    return notFound(res, "User not found")
})

unauthorized(res, message?)

401 
import { unauthorized } from "api-res-kit"

app.post("/login", async (req, res) => {
  const valid = await checkCredentials(req.body)
  if (!valid)
    return unauthorized(res, "Invalid token")
})

forbidden(res, message?)

403 
import { forbidden } from "api-res-kit"

app.get("/admin", (req, res) => {
  if (req.user.role !== "admin")
    return forbidden(res, "Admin access required")
})

validationError(res, errors)

422 
import { validationError } from "api-res-kit"

app.post("/register", (req, res) => {
  validationError(res, [
    { field: "email", message: "Required" },
    { field: "password", message: "Min 8 chars" }
  ])
})

Framework Adapters

Works with your stack

Drop-in adapters for the most popular Node.js frameworks.
One import, zero config — just plug in and go.

Express

Middleware

Register once with app.use() — every route gets helper methods on res automatically.

import { apiResponseMiddleware }
  from "api-res-kit/adapters/express"

app.use(apiResponseMiddleware())

app.get("/users", (req, res) => {
  res.apiSuccess({ data: users })
})

Fastify

Plugin

Register the plugin and all reply objects gain standardized response decorators.

import apiPlugin
  from "api-res-kit/adapters/fastify"

fastify.register(apiPlugin)

fastify.get("/users", (req, reply) => {
  reply.apiSuccess({ data: users })
})

NestJS

Interceptor

Add as a global interceptor — all controllers return consistent response shapes.

import { ApiResponseInterceptor }
  from "api-res-kit/adapters/nestjs"

@Module({
  providers: [{
    provide: APP_INTERCEPTOR,
    useClass: ApiResponseInterceptor
  }]
})
Getting Started

Up and running in 2 minutes

Three simple steps to standardize all your API responses.

1

Install the package

Add api-res-kit to your project via npm, yarn, or pnpm.

$ npm i api-res-kit
2

Import the functions

Import only what you need. Tree-shakable and lightweight.

app.ts 
import {
  success,
  created,
  error,
  notFound,
  validationError,
  paginated
} from "api-res-kit"
3

Use in your routes

Replace inconsistent res.json() calls with standardized responses.

routes/users.ts 
// GET /users — Paginated list
app.get("/users", async (req, res) => {
  const users = await User.find()
  paginated(res, {
    data: users,
    page: 1,
    perPage: 20,
    total: 100
  })
})

// POST /users — Create
app.post("/users", async (req, res) => {
  const errors = validate(req.body)
  if (errors.length)
    return validationError(res, errors)

  const user = await User.create(req.body)
  created(res, {
    data: user,
    message: "User created"
  })
})

// GET /users/:id — Single
app.get("/users/:id", async (req, res) => {
  const user = await User.findById(req.params.id)
  if (!user)
    return notFound(res, "User not found")

  success(res, { data: user })
})
